# Codex operating manual

## Per-session protocol

1. Read `AGENTS.md`, `TASKS.md`, `docs/CODEX_HANDOFF.md` and the current Notion task.
2. Confirm the exact upstream commit.
3. Add/update provenance before copying.
4. Implement one coherent task cluster.
5. Build frequently.
6. Run focused tests, then the project build.
7. Mark complete only with evidence.
8. Leave the repository buildable.
9. Document blockers precisely; never invent an API.
10. End with exact commands, results, provenance and one next task.

## Codex must not

- stop after a plan;
- claim unexecuted tests;
- copy an unverified asset;
- copy code with unresolved lineage;
- bypass transaction safety;
- create a duplicate family graph;
- send GPL code into Immersive EGO;
- silently accept unknown mandatory-mod versions.
